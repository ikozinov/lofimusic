# fcontext
Мы используем утилиту `github.com/ikozinov/fcontext` для того, чтобы ты (AI) имел долгосрочную память и строгий процесс работы с требованиями.
1. ВСЕГДА читай `.fcontext/_README.md` перед началом работы.
2. Требования: Любая новая фича или изменение должно сначала регистрироваться через `fcontext req add "Название"`. Вся работа ведется только после фиксации требования. Завершил задачу — `fcontext req set ID status done`.
3. Аналитика и логи: При завершении сложного рефакторинга или дебага, записывай выводы в `.fcontext/_topics/<name>.md`.
4. Документация: Если нужно проанализировать сторонние файлы (PDF, DOCX), сначала кэшируй их через `fcontext index <file>`, результаты ищи в `_cache/`.
5. Опыт: Используй `fcontext experience` для импорта и экспорта знаний предметной области.
